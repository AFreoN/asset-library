HOW TO MAKE YOUR NEXT RELEASE:
================================================================================

Simple 5-step process:

1. Update package.json version number
   Example: "version": "0.2.0"

2. Update CHANGELOG.md at the top with changes
   Format:
   ## [0.2.0] - 2025-01-15
   ### Added
   - Feature 1
   - Feature 2
   ### Fixed
   - Bug 1

3. Commit changes:
   git add package.json CHANGELOG.md
   git commit -m "Release v0.2.0"
   git push origin main

4. Create and push a git tag:
   git tag -a v0.2.0 -m "Release version 0.2.0"
   git push origin v0.2.0

5. Done! Check GitHub Releases page for automatic release

GitHub will automatically:
✓ Detect your new tag
✓ Extract the changelog from CHANGELOG.md
✓ Create a Release page with changelog as description
✓ Make it available for download
